## 🧱 一、基础架构术语（Transformer 核心）

| 术语 | 英文/缩写 | 核心定义 | 架构师关注点 |
|------|----------|---------|-------------|
| **Transformer** | - | 基于自注意力机制的序列建模架构，大模型基石 | 理解计算复杂度 O(n²) 对长序列/显存的影响 |
| **Self-Attention** | - | 序列内每个位置与其他位置计算注意力权重 | 通信密集操作，分布式训练时需优化 AllReduce |
| **Multi-Head Attention** | MHA | 并行多组注意力头，学习不同语义子空间 | 头数×维度影响显存带宽，需算子融合优化 |
| **Positional Encoding** | PE | 注入序列顺序信息的编码（绝对/相对位置） | RoPE 等相对编码提升长文本外推能力，昇腾需定制算子 |
| **Feed-Forward Network** | FFN | 位置独立的两层 MLP，占参数量 2/3 | 量化/剪枝主要对象，关键层需保留高精度 |
| **LayerNorm** | LN | 层归一化，稳定训练过程 | NPU 上 LN+Activation 融合可提升 15%+ 性能 |
| **Residual Connection** | - | 残差连接，缓解梯度消失 | 深度网络必需，但增加显存访问开销 |
| **Vocabulary** | Vocab | 模型词表，决定 token 化方式 | 词表大小影响 embedding 层显存，中文需优化分词 |

## 🔄 二、训练范式术语

### 预训练阶段
| 术语 | 英文/缩写 | 核心定义 | 工程价值 |
|------|----------|---------|---------|
| **Pre-training** | - | 在海量无标注数据上学习通用表征 | 决定模型基础能力，数据质量 > 数量 |
| **Next Token Prediction** | NTP | 预测序列下一个 token 的自监督任务 | 大模型核心损失函数，隐式学习世界知识 |
| **Causal Language Modeling** | CLM | 仅用历史上下文预测未来（自回归） | GPT 系列基础，推理时天然支持流式生成 |
| **Masked Language Modeling** | MLM | 随机 mask 部分 token 并预测（双向） | BERT 系列基础，适合理解类任务 |
| **Scaling Laws** | - | 损失≈A/N^α + B/D^β + C，指导资源投入 | 架构师核心决策依据：算力/数据/参数权衡 |

### 微调与对齐阶段
| 术语 | 英文/缩写 | 核心定义 | 关键要点 |
|------|----------|---------|---------|
| **Supervised Fine-Tuning** | SFT | 用指令 - 响应对微调，对齐任务格式 | 数据质量关键，1k 高质量 > 100k 噪声数据 |
| **Reinforcement Learning from Human Feedback** | RLHF | 用人类偏好奖励信号优化模型 | 三阶段：奖励模型→PPO 优化→策略迭代 |
| **Proximal Policy Optimization** | PPO | 带 KL 约束的策略梯度算法，防止训练崩溃 | 工程复杂度高，需稳定奖励模型 + 超参调优 |
| **Direct Preference Optimization** | DPO | 直接从偏好对优化策略，无需奖励模型 | 简化对齐流程，训练更稳定，资源需求低 50%+ |
| **Reward Modeling** | RM | 训练模型预测人类偏好分数 | 偏差会放大，需多标注者 + 对抗样本增强 |

## ⚡ 三、高效微调术语（PEFT）

| 术语 | 英文/缩写 | 核心定义 | 参数效率 | 适用场景 |
|------|----------|---------|---------|---------|
| **Parameter-Efficient Fine-Tuning** | PEFT | 仅更新少量参数实现任务适配 | - | 资源受限/多任务场景首选 |
| **Low-Rank Adaptation** | LoRA | 用低秩矩阵 A×B 近似权重更新 ΔW | ~0.1% | 通用首选，推理零开销 |
| **Quantized LoRA** | QLoRA | 4-bit 量化基座 + 未量化 LoRA 适配器 | ~0.1% + 显存↓75% | 单卡微调 70B 模型 |
| **Prefix Tuning** | - | 在 input 前添加可学习向量引导生成 | ~0.01% | 小数据生成任务 |
| **Adapter** | - | 在 FFN 后插入小瓶颈层（MLP） | ~1-3% | 多任务学习，任务隔离 |
| **IA³** | Infused Adapter by Inhibiting and Amplifying | 学习激活值缩放因子 | ~0.01% | 极致参数效率场景 |
| **Prompt Tuning** | - | 仅优化 soft prompt 向量，冻结主干 | ~0.001% | 零样本/少样本迁移 |

> 💡 **架构师决策**：优先选 LoRA/QLoRA（生态成熟+推理无开销），多任务场景考虑 Adapter，极致效率选 IA³

## 🚀 四、推理优化术语

### 核心加速技术
| 术语 | 英文/缩写 | 核心定义 | 性能收益 | 昇腾适配要点 |
|------|----------|---------|---------|-------------|
| **KV Cache** | - | 缓存历史 Key/Value，避免重复计算 | 推理复杂度 O(n²)→O(n) | 显存碎片管理，参考项目二动态内存池 |
| **PagedAttention** | (vLLM) | 分页管理 KV Cache，类似虚拟内存 | 显存利用率↑30%，支持更大 batch | 需适配 CANN 内存分配器 |
| **Speculative Decoding** | - | 小模型草稿 + 大模型验证，并行生成 | 理论加速 2-4x | 需蒸馏小模型，验证开销需优化 |
| **Continuous Batching** | - | 动态合并不同长度请求，提升吞吐 | TPS↑2-5x | 昇腾需优化请求调度器 |
| **Tensor Parallelism** | TP | 层内切分矩阵计算，多卡并行 | 单卡显存↓，支持大模型 | AllReduce 通信优化关键（参考项目一） |
| **Pipeline Parallelism** | PP | 按层切分模型，流水线执行 | 支持超大模型，但有空泡 | 需平衡微批次大小减少 bubble |

### 量化与压缩
| 术语 | 英文/缩写 | 核心定义 | 精度影响 | 工程要点 |
|------|----------|---------|---------|---------|
| **Post-Training Quantization** | PTQ | 训练后量化，无需重训 | 轻微↓（校准集关键） | 校准集需代表真实分布 |
| **Quantization-Aware Training** | QAT | 训练中模拟量化，精度更高 | 几乎无损 | 训练成本↑，适合关键场景 |
| **Weight-Only Quantization** | W8A16 | 仅权重量化，激活保留高精度 | 精度损失<1% | 您的项目实践：关键层保护 |
| **SmoothQuant** | - | 迁移激活异常值到权重，提升量化效果 | INT8 可行 | 需通道级缩放，NPU 需定制算子 |
| **AWQ/GPTQ** | - | 感知权重量化，保留重要权重精度 | 4-bit 可用 | 校准算法复杂，推理需反量化支持 |

## 🌐 五、多模态术语

| 术语 | 英文/缩写 | 核心定义 | 典型应用 | 您的项目关联 |
|------|----------|---------|---------|-------------|
| **Cross-Modal Alignment** | - | 对齐不同模态到统一语义空间 | 图文检索、内容审核 | 项目二音乐多模态的核心挑战 |
| **Contrastive Learning** | - | 拉近匹配对、推远不匹配对的表征学习 | CLIP、零样本分类 | 多模态预训练基础 |
| **InfoNCE Loss** | - | 对比学习核心损失：-log[exp(sim/τ)/Σexp] | 模态对齐训练 | 温度系数τ影响收敛稳定性 |
| **Diffusion Transformer** | DiT | 用 Transformer 作为扩散模型骨干 | 文生图/音乐、多模态生成 | 项目二采用 DiT 架构 |
| **Variational Autoencoder** | VAE | 编码 - 解码框架，学习潜在空间分布 | 图像/音频重建、生成 | 项目二中 VAE 解码是瓶颈 |
| **Temporal Attention** | - | 建模时序依赖的注意力机制 | 视频理解、音乐生成 | 项目二 TeaCache 复用历史计算 |
| **Modality Dropout** | - | 训练时随机丢弃模态，提升鲁棒性 | 模态缺失场景 | 工程落地必备，提升实用性 |

## 📊 六、评估指标术语

### 任务效果指标
| 指标 | 适用场景 | 计算方式 | 架构师解读 |
|------|---------|---------|-----------|
| **Perplexity** | PPL | 语言模型困惑度，越低越好 | 预训练质量核心指标，但≠业务效果 |
| **Accuracy / F1** | 分类任务 | 预测正确率 / 精确率召回率调和 | 业务直接指标，需结合混淆矩阵分析 |
| **BLEU / ROUGE** | 生成任务 | n-gram 重叠度，衡量文本相似度 | 参考指标，需人工评估补充 |
| **MOS** | 音频/多模态 | Mean Opinion Score，人工打分 1-5 | 项目二音质评估核心，>4.0 可商用 |
| **CLIP Score** | 图文生成 | CLIP 计算图文相似度 | 自动化评估生成质量，需校准阈值 |

### 工程效率指标
| 指标 | 定义 | 优化目标 | 您的项目实践 |
|------|------|---------|-------------|
| **Throughput** | 每秒处理样本数 (samples/sec) | 最大化 | 项目一吞吐↑4.3x (1620→7100) |
| **Latency (P99)** | 99% 请求的响应时间 | 最小化 | 项目二单步推理↓30% (420→293ms) |
| **GPU/NPU Utilization** | 算力单元利用率 | >80% | 项目一 AI Core↑44% (57%→82%) |
| **Memory Footprint** | 显存/内存占用 | 最小化 | 项目一显存↓30%，支持更大 batch |
| **Cost per Inference** | 单次推理成本 | 最小化 | 量化+批处理+调度联合优化 |

## 🧭 八、架构师决策速查表

### 技术选型决策树
```
业务需求
├─ 任务类型？
│  ├─ 生成 → 选 Causal LM + KV Cache + Speculative Decoding
│  ├─ 分类 → 选 CLM/MLM + LoRA + 量化
│  └─ 多模态 → 选 CLIP/BLIP + Cross-Attention + 模态 dropout
│
├─ 资源约束？
│  ├─ 单卡 <24GB → QLoRA + W8A16 + vLLM
│  ├─ 多卡集群 → DeepSpeed ZeRO-3 + HCCL 优化
│  └─ 边缘部署 → 知识蒸馏 + INT8 + 模型裁剪
│
└─ 国产化要求？
   ├─ 是 → PyTorch-NPU + CANN 8.0+ + ATC 转换
   └─ 否 → 优先 CUDA 生态，保留迁移接口
```

### 风险预警术语
| 术语 | 风险点 | 缓解方案 |
|------|--------|---------|
| **Catastrophic Forgetting** | 微调后丢失通用能力 | 回放缓冲 + 正则化 + 小学习率 |
| **Reward Hacking** | RLHF 中模型钻奖励漏洞 | 多奖励信号 + 人工审核 + KL 约束 |
| **Quantization Collapse** | 量化后精度骤降 | 敏感度分析 + 混合精度 + 校准集优化 |
| **Context Window Overflow** | 长文本超出模型上下文 | RoPE 外推 + 滑动窗口 + 分层摘要 |
| **NPU Operator Missing** | 昇腾缺少某算子支持 | 自定义 Ascend C 算子 / CPU fallback |
