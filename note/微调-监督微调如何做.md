## 🏗️ 一、SFT 核心架构设计

### 1️⃣ 技术选型决策树（架构师视角）

```
业务需求
├─ 数据量 < 5k 且显存受限？
│  └─ 方案：LoRA/QLoRA + 单卡/少卡
├─ 数据量 > 50k 且追求极致效果？
│  └─ 方案：全参数微调 + DeepSpeed ZeRO-3（参考您项目一经验）
├─ 硬件为昇腾 910B 集群？
│  └─ 方案：PyTorch-NPU + CANN 8.0+ + HCCL 通信优化
└─ 多模态任务（文生图/音乐）？
   └─ 方案：DiT/VAE 架构 + 混合精度训练（参考您项目二经验）
```

### 2️⃣ 标准 SFT 流水线
```
数据准备 → 模型加载 → 策略配置 → 训练循环 → 评估验证 → 模型合并 → 部署上线
```

## 🚀 四、训练实施代码（架构级实现）

### 1️⃣ 核心训练脚本（支持 NPU + DeepSpeed）
```python
# sft_train.py
import torch
import torch_npu  # 昇腾适配
from transformers import AutoTokenizer, AutoModelForCausalLM, TrainingArguments
from peft import LoraConfig, get_peft_model
from datasets import load_from_disk
from trl import SFTTrainer

# 1. 模型加载
model = AutoModelForCausalLM.from_pretrained(
    "Qwen/Qwen-7B-Chat",
    torch_dtype=torch.float16,
    device_map="auto"  # 自动映射到 NPU
)
tokenizer = AutoTokenizer.from_pretrained("Qwen/Qwen-7B-Chat", trust_remote_code=True)

# 2. LoRA 配置 (参数高效微调)
lora_config = LoraConfig(
    r=16, lora_alpha=32, target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05, bias="none", task_type="CAUSAL_LM"
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()

# 3. 训练参数
training_args = TrainingArguments(
    output_dir="./sft_output",
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-5,
    num_train_epochs=3,
    fp16=True,
    logging_steps=10,
    save_strategy="epoch",
    deepspeed="./deepspeed_config.json",  # 启用 DeepSpeed
    dataloader_num_workers=4  # 昇腾推荐值
)

# 4. 训练器
dataset = load_from_disk("./hf_dataset")
trainer = SFTTrainer(
    model=model, args=training_args, train_dataset=dataset,
    tokenizer=tokenizer, dataset_text_field="text"
)

# 5. 启动训练
trainer.train()
model.save_pretrained("./sft_output")
```

### 2️⃣ 分布式训练启动命令（昇腾集群）
```bash

# 多机多卡 (参考您项目一 128 卡经验)
```

## ⚡ 五、性能优化策略（结合您的项目经验）

### 1️⃣ 通信优化（参考项目一）
- **Gradient Packing**：减少 AllReduce 通信次数
- **Sequence Parallelism**：长序列训练时降低显存占用
- **HCCL 调优**：昇腾集群通信库参数调优

### 2️⃣ 显存优化（参考项目一/二）
- **ZeRO-3 + Offload**：参数卸载到 CPU，支持更大模型
- **动态显存分配**：避免碎片化（参考项目一动态显存算法）
- **混合精度**：W8A16 量化训练（参考项目二量化经验）

### 3️⃣ 算子融合（参考项目一/二）
- **自定义算子**：Ascend C 开发融合算子，减少内存搬运
- **算子白名单**：避免 NPU 非亲和算子导致 CPU fallback

## ⚠️ 七、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| **显存溢出 (OOM)** | Batch size 过大/序列过长 | 降低 batch_size，启用 ZeRO-3，梯度累积 |
| **训练发散** | 学习率过高/数据噪声 | 降低 lr，warmup 比例增至 0.1，清洗数据 |
| **NPU 利用率低** | 数据加载瓶颈/算子不亲和 | 增加 dataloader_workers，检查算子白名单 |
| **效果过拟合** | 数据单一/训练轮数过多 | 增加数据多样性，早停 (Early Stopping) |
| **推理性能下降** | LoRA 未合并/量化损失 | 合并权重 `merge_and_unload()`，PTQ 量化 |
