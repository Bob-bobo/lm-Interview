# 贡献指南

感谢您对 Awesome-LLM-Learning 项目的兴趣！我们欢迎各种形式的贡献。

## 如何贡献

### 1. 提交内容

#### 学习路线贡献
- 补充或优化学习路径
- 添加新的学习资源
- 修正过时的内容

#### 面试经验贡献
- 分享真实的面试经历（可匿名化处理）
- 添加高频面试题及参考答案
- 整理特定公司的面试流程

#### 实战笔记贡献
- 分享踩坑经验与解决方案
- 提供可复现的代码示例
- 记录性能调优过程

### 2. 贡献规范

#### 文档格式
- 使用 Markdown 格式
- 中文使用中文标点，英文使用英文标点
- 代码块标注语言类型
- 合理使用标题层级（# / ## / ###）

#### 提交信息规范

```
<type>(<scope>): <subject>

类型说明：
- feat: 新功能或新内容
- fix: 修复错误
- docs: 文档更新
- style: 格式调整（不影响功能）
- refactor: 重构
- test: 测试相关
- chore: 其他杂项
```

示例：
```
feat(interview): 添加大模型 Infra 岗位面试经验

- 新增分布式训练相关面试题
- 补充推理优化考察点
```

### 3. 分支管理

| 分支 | 用途 |
|------|------|
| `main` | 主分支，稳定版本 |
| `dev` | 开发分支，整合新内容 |
| `feature/*` | 功能分支 |

### 4. Pull Request 流程

1. Fork 本仓库
2. 从 `dev` 分支创建你的功能分支
3. 完成开发并测试
4. 提交 Pull Request 到 `dev` 分支
5. 等待 Review 并响应反馈

### 5. 内容审核标准

提交的内容将基于以下标准审核：

- ✅ **准确性**：技术内容正确无误
- ✅ **完整性**：包含必要的上下文和说明
- ✅ **可读性**：结构清晰，表达流畅
- ✅ **实用性**：对学习者有实际帮助
- ✅ **原创性**：优先原创内容，引用需标注来源

## 常见问题

**Q: 我不确定我的贡献是否合适，怎么办？**
A: 欢迎先提交 Issue 或 Discussion 与社区讨论。

**Q: 可以提交公司内部的学习资料吗？**
A: 请确保内容可以公开分享，不包含敏感信息。

**Q: 如何添加新的分类目录？**
A: 请先在 Issue 中提议，说明新分类的必要性和内容规划。
